## 네트워크 포트 및 프로세스 확인 명령어 종합

| 명령어 및 옵션 | 주요 용도 | 특징 및 장점 |
| --- | --- | --- |
| **`netstat -tnlp`** | **포트 및 PID 확인** | 가장 널리 쓰이는 표준 명령어. TCP 포트를 점유 중인 프로그램 이름(p) 표시. |
| **`ss -tulpn`** | **최신 소켓 상태 확인** | `netstat`의 최신 대체 도구. 처리 속도가 매우 빠르며 상세한 소켓 정보 제공. |
| **`lsof -i :[Port]`** | **특정 포트 정밀 추적** | "모든 것은 파일이다" 원리에 따라 특정 포트를 연 프로세스의 실행 경로까지 확인 가능. |
| **`ps -ef | grep [PID]`** | **프로세스 상세 정보** | 위 명령어들로 찾은 PID를 통해 실제 실행 중인 명령줄 전체와 소유자 확인. |
| **`nmap -p [Port] [IP]`** | **외부 포트 개방 확인** | 대상 서버의 방화벽이 해당 포트를 외부로 열어두었는지 스캔하는 네트워크 진단 도구. |
| **`telnet [IP] [Port]`** | **간이 접속 테스트** | 특정 포트가 현재 통신 가능한 상태인지 즉각적으로 확인하는 가장 단순한 방법. |

## 포트 연결 상태(State) 판별 기준

| 상태 (State) | 서비스 가능 여부 | 상태 설명 |
| --- | --- | --- |
| **LISTEN** | **YES (대기 중)** | 서버가 클라이언트의 접속 요청을 기다리고 있는 정상적인 서비스 상태입니다. |
| **ESTABLISHED** | **YES (통신 중)** | 클라이언트와 서버가 연결되어 현재 데이터를 주고받고 있는 상태입니다. |
| **TIME_WAIT** | **NO (종료 중)** | 연결 종료 후 잔여 패킷을 기다리는 상태입니다. 잠시 후 자동으로 사라집니다. |
| **CLOSE_WAIT** | **NO (연결 끊김)** | 상대방이 접속을 종료하여 나도 종료를 준비하는 상태입니다. 이 상태가 길면 버그를 의심해야 합니다. |

### 💡 실전 체크리스트 (수정/보완 시 참고)

* **권한 문제:** `netstat`이나 `lsof`에서 프로그램 이름(`-p`)을 보려면 반드시 `sudo` 권한이 필요합니다.
* **0.0.0.0 vs 127.0.0.1:**
* `0.0.0.0` 또는 `*`: 모든 네트워크 카드에서 접속 가능 (외부 서비스 중)
* `127.0.0.1` (localhost): 해당 서버 내부에서만 접속 가능 (외부 차단 상태)


* **IPv6:** `:::80`과 같은 표시는 IPv6 환경에서 80번 포트가 열려 있음을 뜻합니다.
