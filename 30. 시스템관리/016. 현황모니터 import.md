# 그라파나 노드 현황 모니터링 가이드
1. 추천 대시보드 활용

그라파나 커뮤니티의 템플릿을 활용하는 것이 가장 효율적입니다.

• 추천 ID: `1860` (Node Exporter Full)

• 설정 방법: `Dashboards` -> `New` -> `Import` 메뉴에서 ID `1860` 입력

2. 주요 패널 구성 요소

직접 대시보드를 구축할 때 포함해야 할 핵심 지표입니다.

• CPU Usage: `100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)`

• Memory Usage: `node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes`

• Disk Space: `1 - (node_filesystem_avail_bytes / node_filesystem_size_bytes)`

• Node Status: `up{job="node-exporter"}` (1: 정상, 0: 장애)

3. 노드별 필터링 (Variables)

여러 노드 관리를 위해 상단 드롭다운 메뉴를 구성합니다.

• 설정: `Dashboard settings` -> `Variables`

• Query: `label_values(node_cpu_seconds_total, instance)`

4. 실전 운영 팁

1. Node Exporter 필수: 각 노드에 에이전트가 설치되어 있어야 합니다.

2. 알람(Alerting): 임계치 초과 시 슬랙이나 이메일 알림 설정을 권장합니다.

3. 수집 주기: 리소스 효율을 위해 보통 15~30초 주기를 권장합니다.
