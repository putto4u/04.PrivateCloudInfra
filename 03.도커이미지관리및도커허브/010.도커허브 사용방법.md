## 도커 허브(Docker Hub) 레지스트리 활용 가이드

도커 허브는 전 세계에서 가장 큰 컨테이너 이미지 저장소(Registry)로, 개발자가 이미지를 저장(Push)하고 누구나 내려받을 수(Pull) 있는 클라우드 서비스입니다.

---

### 1. 도커 허브 주요 URL 및 정보

도커 허브를 이용하기 위해 반드시 알아야 할 주소들입니다.

* **공식 웹사이트:** [https://hub.docker.com](https://hub.docker.com)
* 회원가입, 이미지 검색, 저장소(Repository) 관리 등을 수행합니다.


* **기본 레지스트리 주소:** `docker.io` (도커 엔진에서 기본값으로 설정됨)
* **이미지 경로 구조:** `docker.io/[계정명]/[저장소명]:[태그]`
* 예: `docker.io/putto4u/git_install_vm:latest`



---

### 2. 도커 허브 기본 사용 흐름

도커 허브를 사용하는 과정은 크게 **로그인 → 이미지 태깅 → 업로드** 순서로 진행됩니다.

#### ① 로그인 (CLI)

터미널에서 도커 허브 계정으로 인증을 수행합니다.

```bash
docker login

```

* 사용자 이름(Username)과 비밀번호(Password)를 입력합니다. (보안을 위해 **Access Token** 사용 권장)

#### ② 이미지 태그 설정 (Tagging)

내 로컬에 있는 이미지를 내 도커 허브 저장소 주소에 맞게 별칭을 붙여야 합니다.

```bash
# 형식: docker tag [로컬_이미지명] [내_계정명]/[저장소명]:[태그]
docker tag my-web-app:1.0 putto4u/my-web-app:1.0

```

#### ③ 이미지 업로드 (Push)

태그가 지정된 이미지를 도커 허브로 전송합니다.

```bash
docker push putto4u/my-web-app:1.0

```

#### ④ 이미지 다운로드 (Pull)

업로드된 이미지는 어디서든 아래 명령어로 내려받을 수 있습니다.

```bash
docker pull putto4u/my-web-app:1.0

```

---

### 3. 공식 이미지와 사용자 이미지 구분

| 구분 | URL/이름 형식 | 특징 |
| --- | --- | --- |
| **공식 이미지** | `docker.io/library/nginx` (또는 `nginx`) | 도커가 직접 관리하며 보안/신뢰도가 높음 |
| **사용자 이미지** | `docker.io/putto4u/nginx` | 개인이 수정하여 배포한 커스텀 이미지 |

---

### 4. 주요 관리 팁

* **Private Repository:** 무료 계정은 보통 1개의 비공개 저장소만 제공합니다. (중요 소스코드 포함 시 주의)
* **Automated Builds:** GitHub와 연동하여 소스 푸시 시 자동으로 이미지가 빌드되도록 설정 가능합니다.
* **Search:** 터미널에서도 `docker search [키워드]` 명령어로 이미지를 검색할 수 있습니다.

---

### ⚠️ 비용 안내 (AWS 및 트래픽)

* **Docker Hub:** 무료 티어의 경우 **6시간당 200회(인증 시)**의 Pull 제한이 있습니다.
* **데이터 비용:** AWS EC2에서 외부 도커 허브로 이미지를 올리거나 내릴 때 발생하는 **데이터 전송 비용(Data Transfer Out)**은 AWS 과금 체계를 따르므로 대용량 이미지 처리 시 주의하십시오.
