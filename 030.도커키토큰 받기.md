도커 허브(Docker Hub)에 로그인하거나 이미지를 푸시(Push)할 때, 실제 계정 비밀번호 대신 사용하는 **액세스 토큰(Access Token)** 발급 방법입니다. 보안을 위해 토큰 사용이 강력히 권장됩니다.

---

### 도커 허브 액세스 토큰 발급 과정

1. **도커 허브 로그인**: [Docker Hub 공식 사이트](https://hub.docker.com/)에 접속하여 로그인합니다.
2. **프로필 설정 진입**: 오른쪽 상단 프로필 아이콘을 클릭한 뒤 **[Account Settings]** 메뉴를 선택합니다.
3. **Security 메뉴 선택**: 왼쪽 사이드바 메뉴에서 **[Security]** 탭을 클릭합니다.
4. **토큰 생성**: **[New Access Token]** 버튼을 클릭합니다.
5. **토큰 상세 설정**:
* **Access Token Description**: 토큰의 용도를 입력합니다. (예: `ubuntu-server-token`)
* **Access Permissions**: 권한 범위를 설정합니다. (보통 이미지를 올리고 내려받기 위해 `Read, Write, Delete` 또는 `Read & Write`를 선택합니다.)


6. **토큰 복사**: 생성된 토큰 값을 **반드시 따로 저장(복사)**해 두세요. 화면을 벗어나면 다시 확인할 수 없으므로 안전한 곳에 보관해야 합니다.

---

### 터미널에서 토큰으로 로그인하기

발급받은 토큰은 CLI(터미널) 환경에서 비밀번호 대신 사용합니다.

```bash
# 1. 로그인 명령 입력
docker login -u [도커허브_사용자ID]

# 2. Password 입력 요청 시, 복사해둔 '액세스 토큰'을 붙여넣기 합니다.

```

로그인에 성공하면 `Login Succeeded` 메시지가 출력됩니다.

---

### 토큰 사용의 장점 및 관리

* **보안 강화**: 실제 비밀번호가 노출되는 것을 방지합니다.
* **권한 제어**: 특정 용도로만 사용하도록 권한을 제한할 수 있습니다.
* **간편한 폐기**: 특정 서버나 환경이 해킹되었을 때 해당 토큰만 즉시 삭제(Revoke)하여 보안 사고를 막을 수 있습니다.

> [!IMPORTANT]
> **과금 및 할당량 주의**: 도커 허브는 무료 계정의 경우 **이미지 풀(Pull) 횟수 제한(Rate Limit)**이 있습니다. 익명 사용자는 100회/6시간, 로그인 사용자는 200회/6시간으로 제한됩니다. AWS EC2 등에서 대규모 배포를 진행할 때 이 제한에 걸리면 배포가 중단될 수 있으니, 로그인 후 사용하는 것이 유리합니다.
