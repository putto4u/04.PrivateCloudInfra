## 우리 학습 훈련 목표  
현업에서 가장 많이 구축하고 있는 웹서비스를 하기위한 인프라를 구축하는 것에 있다. 
1. 인프라를 구축하는 방법은 전통적인 방법(전산실에서 실물/물리적으로 구축하는 방법 - 온프레미스)으로 구축하는 훈련을 한다.
2. 인프라를 Iac(인프라를 물리적 구축 이전에 코드(문서화)로 만들어서 자동으로 구축하게 하는 방법이 있다.
3. 1번째는 앤서블 툴로 소규모의 프로그램들을 배포(설치)하는 훈련을 한다. 
4. 2번째로 도커 툴로 프로그램들을 컨테이너로 만들어서 배포(설치)하는 훈련을 한다.
5. 컨테이너 이미지로 배포된 인프라의 규모가 커지면 관리에 어려움이 있다.
6. 3번째. 이에대한 해결책으로 나온 K8s(쿠버네티스) 툴을 이용하여 대규모 인프라를 운영하는 방법을 훈련한다.
7. 모든 실습의 마지막 결과는   웹서버설치-> DB서버설치-> 두 서버 연결 -> 스토리지 설치 -> 전체 웹서비스 접속 확인하는 것이다.
8. 실무 현장의 인프라와 가깝게 좀 더 디테일한 인프라 구축을 훈련한다.
   
10. 1단계 : 웹서버 엔진을 설치 --------------------------------프론트 웹서버엔진을 설치하고 브라우저로 서비스를 확인한다. 
11. 2단계 : DB서버 엔진 설치   ------------------------------- 가능하다면 데이터베이스 데이터를 주입(복원)을 경험한다.
12. 3단계 : 백엔드 웹서버 설치 ------------------------------- 백앤드서버를 설치하고  브라우저로 서비스를 확인한다.
13. 4단계 : 프론트 웹서버와 백앤드 웹서버를 연결한다. --------- URi로 서로 다른 웹페이지가 보이도록 하는 것을 경험한다.
14. 5단계 : 백앤드 서버와 DB서버를 연결한다.    --------------- 실제 각종 사이트에서 제공하는 동적콘텐츠 서비스의 인프라 구축을 경험한다.
15. 6단계 : 실제 데이터 사례를 DB서버에 주입하여 서비스 ------- DevOps 환경에서 개발팀이 웹 서비스를 만들어 인프라와 함께 작동하는 경험을 한다.
16. 7단계 : 스토리지/파일서버를 설치  ------------------------ 모든 서비스에 필요한 스토리지를 구축하고 파일서버 서비스를 경험한다.

17. 4번째 IT서비스가 자유롭게 확장되고 관리되는 클라우드를 경험한다.
18. 아마존 퍼블릭 클라우드에서 위에서 구축했던 웹서비스(비지니스)를 할수 있는 인프라를 구축해본다.
19. 아마존 클라우드에서 제공하는 서비스 중 중요 서비스에 대해 이해하고 경험해 본다.

20. 이 모든 인프라 구축 과정은 최종적으로 IaC (코드화-문서화)하고, 해당 코드로도 자동 구축할 수 있도록 훈련한다.

##   모의 해킹과 보안
1. 구축된 서비스 인프라를 모의 해킹   -----------------------------  구축된 인프라를 보호하는 보안 전문가로서 해킹을 실습해보고 이해한다.
2. 각종 해킹 기술과 툴 사용          ----------------------------- 여러가지 해킹 기법과 툴들을 사용하여 인프라 보안 대책과 방법을 이해한다.
3. 모의 해킹 결과 분석         ------------------------------- 해킹 결과 분석 방법과 대응 방법에 대하여 이해한다.
4. 보안 대응 및 보안 시스템 구축   --------------------------- 해킹과 보안 시스템 전체를 이해하고 보안 시스템 구축에 대한 방안 및 기술을 경험한다.
