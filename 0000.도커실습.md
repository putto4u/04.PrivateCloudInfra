도커의 핵심 기능인 이미지 빌드, 컨테이너 실행, 포트 포워딩, 그리고 도커허브 공유까지 한 번에 익힐 수 있는 실습 10선을 정리했습니다. 각 실습은 브라우저 확인이 가능한 앱 위주로 구성했습니다.

---

## 도커 이미지 빌드 및 배포 실습 10선

### 1. Apache 웹 서버 커스텀 페이지 실행

가장 기본적인 웹 서버 실행 실습입니다.

* **목표**: `index.html`을 포함한 Apache 이미지 빌드 및 실행
* **핵심 명령어**: `FROM httpd:latest`, `COPY ./index.html /usr/local/apache2/htdocs/`
* **브라우저 확인**: `http://localhost:80`

### 2. Nginx를 이용한 정적 사이트 호스팅

Nginx의 기본 경로를 익히고 실행합니다.

* **목표**: Nginx 기본 페이지를 자신만의 HTML로 교체
* **핵심 명령어**: `docker run -d -p 8080:80 --name my-nginx nginx`
* **브라우저 확인**: `http://localhost:8080`

### 3. Python Flask "Hello World" 앱

경량 웹 프레임워크인 Flask를 이미지화합니다.

* **목표**: Python 소스 코드를 컨테이너 내부에서 실행
* **핵심 명령어**: `WORKDIR /app`, `RUN pip install flask`, `CMD ["python", "app.py"]`
* **브라우저 확인**: `http://localhost:5000`

### 4. Django 프레임워크 초기 페이지 띄우기

복잡한 파이썬 프레임워크의 환경 구축을 자동화합니다.

* **목표**: `manage.py runserver`를 Dockerfile로 구성
* **핵심 명령어**: `EXPOSE 8000`, `python manage.py runserver 0.0.0.0:8000`
* **브라우저 확인**: `http://localhost:8000`

### 5. MySQL 데이터베이스 컨테이너 구동

환경 변수(`-e`)를 사용하여 DB 설정을 주입합니다.

* **목표**: Root 비밀번호를 설정하고 DB 컨테이너 실행
* **핵심 명령어**: `docker run -e MYSQL_ROOT_PASSWORD=pw -d mysql:8.0`
* **확인**: `docker exec -it mysql_con mysql -u root -p`

### 6. Node.js Express 웹 서버 구축

자바스크립트 런타임 환경을 컨테이너화합니다.

* **목표**: `package.json` 의존성 설치 및 서버 구동
* **핵심 명령어**: `RUN npm install`, `npm start`
* **브라우저 확인**: `http://localhost:3000`

### 7. PHP + Apache 결합 환경 (LAMP 기초)

PHP 스크립트가 해석되는 환경을 구축합니다.

* **목표**: `phpinfo()` 페이지를 브라우저에 출력
* **핵심 명령어**: `FROM php:7.4-apache`, `COPY src/ /var/www/html/`
* **브라우저 확인**: `http://localhost:80/info.php`

### 8. WordPress 블로그 즉시 실행

애플리케이션 이미지를 내려받아 즉시 서비스합니다.

* **목표**: 도커허브의 공식 워드프레스 이미지 활용
* **핵심 명령어**: `docker pull wordpress`, `docker run --name my-wp -p 80:80 -d wordpress`
* **브라우저 확인**: `http://localhost/wp-admin`

### 9. 실습 이미지 도커허브(Docker Hub)에 업로드

자신이 만든 이미지를 전 세계에 공유합니다.

* **목표**: `docker push` 과정 숙달
* **핵심 과정**:
1. `docker login`
2. `docker tag local-image:v1 username/repo:v1`
3. `docker push username/repo:v1`



### 10. 타인(또는 본인)의 이미지 다운로드 및 실행

다른 환경에서 이미지를 가져와 동일하게 실행되는지 확인합니다.

* **목표**: `docker pull` 후 실행하여 이식성 증명
* **핵심 명령어**: `docker run -p 80:80 username/repo:v1`
* **확인**: 다른 PC나 서버에서 브라우저 접속 테스트

## 실습 흐름도

| 단계 | 주요 작업 | 명령어 예시 |
| --- | --- | --- |
| **Build** | 소스 코드와 환경을 이미지로 생성 | `docker build -t app:v1 .` |
| **Ship** | 이미지를 레지스트리에 저장 | `docker push user/app:v1` |
| **Run** | 어디서든 이미지를 받아 실행 | `docker run -d -p 80:80 app:v1` |

> **AWS 과금 및 리소스 관리**:
> * 위 실습을 AWS EC2에서 진행할 경우, 여러 컨테이너를 띄우면 메모리 부족(OOM)이 발생할 수 있습니다.
