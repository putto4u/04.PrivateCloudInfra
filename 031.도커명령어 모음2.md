도커 환경을 관리하며 가장 빈번하게 사용하게 되는 명령어들을 기능별로 분류하여 실제 사례와 함께 정리했습니다. 이 표는 실무에서 바로 참고할 수 있도록 주요 옵션을 포함하고 있습니다.

---

## 1. 이미지 관리 (Image Management)

이미지를 내려받고, 생성하고, 삭제할 때 사용하는 명령어입니다.

| 명령어 | 실제 활용 사례 | 설명 |
| --- | --- | --- |
| **build** | `docker build -t my-app:v1 .` | 현재 디렉터리(`.`)의 Dockerfile로 `my-app:v1` 이미지 생성 |
| **images** | `docker images` | 로컬에 저장된 모든 이미지 목록 출력 |
| **pull** | `docker pull nginx:latest` | Docker Hub에서 최신 Nginx 이미지를 로컬로 내려받음 |
| **rmi** | `docker rmi -f $(docker images -q)` | 사용 중이지 않은 이미지와 상관없이 로컬의 모든 이미지 강제 삭제 |
| **tag** | `docker tag my-app:v1 my-repo/my-app:v1` | 기존 이미지에 새로운 저장소 이름이나 태그를 부여 |

---

## 2. 컨테이너 실행 및 제어 (Container Lifecycle)

컨테이너를 생성하고 실행 상태를 관리하는 핵심 명령어입니다.

| 명령어 | 실제 활용 사례 | 설명 |
| --- | --- | --- |
| **run** | `docker run -d -p 80:80 --name web nginx` | 백그라운드(`-d`)로 포트 포워딩(`-p`)을 설정해 컨테이너 실행 |
| **ps** | `docker ps -a` | 중지된 컨테이너를 포함하여 전체 목록 확인 |
| **stop** | `docker stop web_con1` | 실행 중인 특정 컨테이너를 안전하게 중지 |
| **start** | `docker start web_con1` | 중지되어 있는 컨테이너를 다시 실행 |
| **rm** | `docker rm -f web_con1` | 실행 중인 컨테이너를 강제(`-f`)로 종료하고 즉시 삭제 |

---

## 3. 컨테이너 내부 확인 및 디버깅 (Inspection & Debug)

실행 중인 컨테이너 내부를 살피거나 문제를 진단할 때 필수적입니다.

| 명령어 | 실제 활용 사례 | 설명 |
| --- | --- | --- |
| **exec** | `docker exec -it web_con1 /bin/bash` | 실행 중인 컨테이너 내부 터미널에 대화형(`-it`)으로 접속 |
| **logs** | `docker logs -f web_con1` | 컨테이너 내부 로그를 실시간(`-f`)으로 확인 |
| **inspect** | `docker inspect web_con1` | 컨테이너의 IP 주소, 설정 등 상세 네트워크 정보를 JSON 형태로 출력 |
| **cp** | `docker cp ./index.html web_con1:/tmp/` | 로컬의 파일을 실행 중인 컨테이너 내부로 복사 |
| **top** | `docker top web_con1` | 컨테이너 내부에서 실행 중인 프로세스 목록 확인 |

---

## 4. 자원 및 네트워크 관리 (System Management)

시스템의 전체적인 상태를 파악하고 정리할 때 사용합니다.

| 명령어 | 실제 활용 사례 | 설명 |
| --- | --- | --- |
| **stats** | `docker stats` | 컨테이너별 CPU, 메모리 사용량을 실시간 스트리밍으로 확인 |
| **prune** | `docker system prune -a` | 중지된 컨테이너, 사용되지 않는 네트워크, 이미지를 모두 삭제하여 용량 확보 |
| **volume ls** | `docker volume ls` | 생성된 도커 볼륨(데이터 저장소) 목록 확인 |
| **network ls** | `docker network ls` | 생성된 도커 가상 네트워크 목록 확인 |

---

자주 사용하는 도커 명령어들을 표로 정리했습니다. 실무에서는 특히 `docker ps -a`와 `docker logs -f`를 조합하여 컨테이너 상태를 점검하는 것이 기본입니다.
