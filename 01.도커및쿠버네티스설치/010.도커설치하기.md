우분투(Ubuntu) 환경에서 도커(Docker)를 설치하고 정상 작동 여부를 테스트하는 과정입니다. 공식 저장소를 사용하는 가장 안정적인 방식을 기준으로 정리했습니다.

---

### 필수 패키지 설치 및 저장소 설정

도커를 설치하기 전, 시스템을 업데이트하고 HTTPS를 통해 저장소를 사용할 수 있도록 필요한 패키지들을 먼저 설치합니다.

```bash
# 1. 시스템 패키지 업데이트
sudo apt update

# 2. 필수 패키지 설치
sudo apt install -y ca-certificates curl gnupg lsb-release

```

### 도커 공식 GPG 키 및 저장소 추가

도커 패키지의 보안 인증을 위해 공식 GPG 키를 추가하고 저장소를 등록합니다.

```bash
# 1. GPG 키 저장을 위한 디렉토리 생성 및 키 추가
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 2. 도커 저장소(Repository) 등록
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

```

### 도커 엔진 설치

저장소 설정이 완료되면 실제 도커 엔진과 관련 도구들을 설치합니다.

```bash
# 1. 저장소 정보 갱신
sudo apt update

# 2. 도커 엔진, CLI, 컨테이너디, 플러그인 설치
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

```

> [!IMPORTANT]
> **과금 주의**: 도커 엔진 자체는 오픈소스(Community Edition)로 무료이나, 기업 환경에서 **Docker Desktop**을 사용할 경우 사용자 수 및 매출 규모에 따라 유료 구독이 필요할 수 있습니다. 리눅스 서버용 도커 엔진은 무료로 이용 가능합니다.

---

### 설치 확인 및 테스트

설치가 완료되었다면 서비스를 확인하고 테스트용 컨테이너를 실행해 봅니다.

1. **서비스 상태 확인**:
```bash
sudo systemctl status docker

```


`Active: active (running)` 상태가 보이면 정상입니다.
2. **Hello-World 테스트**:
```bash
sudo docker run hello-world

```


"Hello from Docker!"라는 메시지가 출력되면 도커가 이미지를 풀(Pull)하고 컨테이너를 실행하는 모든 과정이 정상적으로 작동하는 것입니다.

---

### (선택) sudo 없이 도커 사용하기

매번 `sudo`를 붙이는 것이 번거롭다면 현재 사용자를 `docker` 그룹에 추가하세요.

```bash
# 1. 현재 사용자를 docker 그룹에 추가
sudo usermod -aG docker $USER

# 2. 변경 사항 적용 (로그아웃 후 다시 로그인하거나 아래 명령 실행)
newgrp docker

```
